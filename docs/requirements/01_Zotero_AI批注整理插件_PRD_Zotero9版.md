# Zotero AI 批注整理插件产品需求文档（PRD）
> 兼容性说明：本文档以 Zotero 9.x 为目标版本。具体实现时，所有菜单、Item Pane、PDF Reader、批注读取和插件生命周期 API 都必须以当前 Zotero 9 官方文档、示例插件和实际类型定义为准，不得仅通过将旧版文档中的版本号替换为 9 来判断兼容性。

> 产品暂定名：**Zotero AI Notes / Zotero 智读笔记**  
> 文档版本：V1.0  
> 产品形态：Zotero 9 桌面端插件  
> 核心定位：将用户在 PDF 阅读过程中产生的高亮、批注和思考，自动整理为结构化 Markdown 笔记，并进一步生成可编辑思维导图。

---

## 1. 产品概述

### 1.1 产品背景

Zotero 已经具备较完整的 PDF 阅读、文本高亮、区域批注、标签、文献笔记和 Markdown 导出能力，但当前用户仍需要手动完成以下工作：

1. 将分散在不同页码中的批注重新归类；
2. 区分原文摘录、个人思考、疑问和行动项；
3. 删除重复内容并补充上下文；
4. 将批注整理为结构完整的学习笔记；
5. 根据笔记手动制作思维导图；
6. 将结果保存回 Zotero，或导出到 Obsidian、Typora 等 Markdown 工具。

现有 Zotero AI 插件大多强调“让 AI 阅读整篇论文并生成摘要”，但较少真正围绕用户已经完成的阅读行为进行整理。

本产品希望解决的问题不是“替用户读 PDF”，而是：

> **把用户真正读过、划过、想过的内容，整理成可复习、可追溯、可继续加工的个人知识资产。**

---

## 2. 产品目标

### 2.1 核心目标

用户完成 PDF 阅读和批注后，可以在 Zotero 中一键完成：

```text
读取批注
→ AI 分类与重组
→ 生成结构化 Markdown 笔记
→ 用户预览和修改
→ 写回 Zotero
→ 导出 Markdown
→ 生成思维导图
```

### 2.2 产品价值

#### 对普通学习者

- 减少手动复制和整理批注的时间；
- 将零散摘录转化为逻辑完整的学习笔记；
- 方便在 Obsidian 等知识管理工具中继续使用；
- 帮助回顾课程资料、电子书和技术文档。

#### 对科研人员

- 快速形成论文精读笔记；
- 保留页码、引用和原文跳转链接；
- 支持多篇论文对比与文献综述；
- 将方法、实验、结论和不足自动分类。

#### 对 AI 产品经理和职场用户

- 将行业报告、产品资料和研究论文整理为知识卡片；
- 自动提取产品启发、应用场景、风险和行动项；
- 生成适合汇报和复习的思维导图；
- 形成可持续积累的个人知识库。

## 3. 产品定位

### 3.1 一句话定位

一款以用户批注为中心的 Zotero AI 笔记整理插件。

### 3.2 核心差异化

传统 PDF AI 工具通常采用：

```text
AI 读取整篇 PDF
→ 自动总结
```

本产品采用：

```text
用户先阅读和批注
→ AI 理解用户关注点
→ 根据高亮、批注、标签和页码重新组织
```

产品强调：

- AI 不替代阅读；
- AI 不编造未出现的内容；
- 用户批注优先于通用摘要；
- 所有结论都可以追溯到原文页码；
- 生成结果必须经过用户确认后写回文献库。

---

## 4. 目标用户

### 4.1 核心用户

1. 经常使用 Zotero 阅读论文的学生和科研人员；
2. 使用 Zotero 管理课程 PDF、电子书和技术资料的学习者；
3. 使用 Obsidian、Logseq 或 Typora 管理 Markdown 笔记的用户；
4. 希望通过 AI 整理个人阅读成果的知识工作者；
5. 需要快速生成学习笔记和思维导图的 AI 产品经理。

### 4.2 典型用户画像

#### 用户 A：研究生

- 每周阅读 5～10 篇论文；
- 会对方法、实验结果和不足做高亮；
- 阅读后需要写论文精读笔记；
- 希望保留引用页码和原文链接。

#### 用户 B：AI 产品经理

- 经常阅读 AI 行业报告和技术论文；
- 会记录产品启发、场景和风险；
- 需要将资料整理为结构化 Markdown；
- 希望生成思维导图用于学习和汇报。

#### 用户 C：知识管理爱好者

- 使用 Zotero 阅读，使用 Obsidian 管理知识；
- 不希望手动复制大量高亮内容；
- 希望自动同步 Markdown 文件；
- 更关注本地存储和数据隐私。

## 5. 核心使用场景

### 5.1 单篇 PDF 批注整理

用户阅读完一篇 PDF，在文献上点击“使用 AI 整理批注”，插件读取该文献的全部高亮和评论，生成结构化 Markdown 笔记。

### 5.2 论文精读笔记

插件将批注按照以下结构整理：

- 研究背景；
- 要解决的问题；
- 核心方法；
- 数据集与实验；
- 主要结论；
- 论文局限；
- 用户自己的思考；
- 后续可研究问题。

### 5.3 课程资料学习笔记

插件将教材或课程讲义的批注整理为：

- 本章目标；
- 核心概念；
- 原理解释；
- 案例；
- 易错点；
- 我的理解；
- 尚未解决的问题；
- 复习清单。

### 5.4 产品经理视角整理

插件将技术论文或行业报告整理为：

- 技术解决了什么问题；
- 核心能力；
- 适用场景；
- 产品价值；
- 用户体验影响；
- 商业化可能；
- 实施风险；
- 对 AI 产品经理的启发。

### 5.5 思维导图生成

基于整理后的笔记生成层级化思维导图，可预览、复制或导出。

### 5.6 多篇文献对比

用户选择多篇文献后，插件根据批注生成横向对比：

- 研究问题；
- 核心方法；
- 模型或技术路线；
- 数据集；
- 实验指标；
- 优势；
- 局限；
- 适用场景。

## 6. 产品原则

1. **以用户批注为核心**：默认不对未批注的全文进行大规模总结。
2. **事实可追溯**：尽可能保留页码和 Zotero 原文跳转链接。
3. **原文与个人观点分离**：明确区分高亮原文、用户评论和 AI 归纳。
4. **用户最终确认**：AI 结果先进入预览区，用户确认后再写回 Zotero。
5. **不覆盖原始数据**：不修改用户已有高亮和批注。
6. **支持多模型**：兼容 OpenAI、Claude、Gemini、DeepSeek、通义千问、Kimi、Ollama 等。
7. **隐私优先**：向用户明确展示哪些内容会被发送给模型。
8. **渐进式开发**：先完成单篇批注整理，再扩展全文、多篇对比和高级知识管理。

## 7. 核心业务流程

### 7.1 标准流程

```text
用户选择 Zotero 文献
        ↓
点击“AI 整理批注”
        ↓
插件检查是否存在 PDF 和批注
        ↓
读取文献元数据、批注、标签、页码和评论
        ↓
用户选择笔记模板和模型
        ↓
插件构造结构化请求
        ↓
调用大模型
        ↓
返回 Markdown 笔记和思维导图结构
        ↓
用户预览、编辑和重新生成
        ↓
保存为 Zotero 子笔记
        ↓
导出 Markdown / 思维导图
```

### 7.2 无批注场景

当文献没有任何批注时，提示：

> 当前文献尚无批注。请先阅读并添加高亮或评论，或者选择“AI 阅读全文”模式。

“AI 阅读全文”不纳入第一阶段 MVP，可在后续版本中实现。

### 7.3 生成失败场景

出现模型接口异常、余额不足、超时或返回格式错误时：

- 不删除已有内容；
- 显示明确错误原因；
- 支持重新生成；
- 支持切换模型；
- 保留本次输入内容；
- 可复制原始请求用于排查问题。

## 8. 功能需求总览

| 模块 | 功能 | 优先级 |
|---|---|---:|
| 文献读取 | 获取文献元数据 | P0 |
| 批注读取 | 获取高亮、评论、页码、标签、颜色 | P0 |
| AI 整理 | 按模板整理为 Markdown | P0 |
| 预览编辑 | 预览、修改、重新生成 | P0 |
| Zotero 写回 | 创建新的子笔记 | P0 |
| Markdown 导出 | 导出本地 `.md` 文件 | P0 |
| 模型配置 | OpenAI-compatible API 配置 | P0 |
| 模板管理 | 内置多种笔记模板 | P1 |
| 思维导图 | 生成 Markmap 或 Mermaid | P1 |
| Obsidian 集成 | 指定文件夹导出 | P1 |
| 多篇对比 | 对多篇文献批注做横向整理 | P2 |
| 本地模型 | 支持 Ollama | P2 |

# 9. 分阶段开发计划

## 阶段一：MVP——单篇批注生成 Markdown 笔记

### 9.1 阶段目标

验证核心价值：

> 用户能否通过一个按钮，把当前文献的全部批注整理成一份结构化 Markdown 笔记。

### 9.2 必做功能

#### 9.2.1 插件基础框架

- 基于 Zotero 9 插件机制搭建项目；
- 支持在 Zotero 9 中安装、启用、禁用和卸载；
- 在文献右键菜单增加“AI 整理批注”；
- 在 Zotero 设置中增加插件配置页；
- 支持中文和英文基础界面。

#### 9.2.2 当前文献识别

插件能够识别：

- 当前选中的文献条目；
- 文献标题；
- 作者；
- 出版年份；
- 摘要；
- PDF 附件；
- 是否存在批注。

#### 9.2.3 批注读取

读取当前文献下所有 PDF 批注，包括：

- 高亮原文；
- 文本评论；
- 页码；
- 批注颜色；
- 批注标签；
- 批注类型；
- Zotero 原文跳转链接；
- 批注创建时间和修改时间。

#### 9.2.4 数据结构化

将批注转换为统一 JSON：

```json
{
  "item": {
    "title": "文献标题",
    "authors": ["作者 A", "作者 B"],
    "year": "2026",
    "abstract": "摘要"
  },
  "annotations": [
    {
      "type": "highlight",
      "page": 3,
      "color": "yellow",
      "text": "高亮原文",
      "comment": "用户批注",
      "tags": ["核心观点"],
      "link": "zotero://open-pdf/..."
    }
  ]
}
```

#### 9.2.5 模型配置

MVP 先支持 OpenAI-compatible 接口：

- Base URL；
- API Key；
- 模型名称；
- Temperature；
- 最大输出长度；
- 测试连接；
- 保存配置。

首版可兼容：

- OpenAI；
- DeepSeek；
- 通义千问兼容接口；
- Kimi 兼容接口；
- 其他 OpenAI-compatible 服务。

#### 9.2.6 内置笔记模板

MVP 至少提供三个模板：

1. 通用学习笔记；
2. 论文精读笔记；
3. AI 产品经理视角笔记。

用户每次生成前可以选择模板。

#### 9.2.7 AI 生成

模型输出 Markdown，至少包括：

- 文献信息；
- 核心主题；
- 结构化内容；
- 原文引用；
- 用户批注；
- 核心结论；
- 尚未解决的问题。

要求：

- 不编造原文未出现的事实；
- 原文引用必须来自输入批注；
- 尽量保留页码和原文链接；
- 重复批注应合并；
- 按主题组织，而不是机械按页码排列。

#### 9.2.8 结果预览

生成完成后显示预览窗口：

- 左侧：生成设置；
- 中间：Markdown 编辑器；
- 右侧：渲染预览；
- 支持修改文本；
- 支持复制 Markdown；
- 支持重新生成；
- 支持取消。

#### 9.2.9 写回 Zotero

用户确认后：

- 在当前文献下创建新的子笔记；
- 笔记标题默认命名为“AI 整理笔记 - 日期”；
- 不覆盖已有笔记；
- 保存生成时间、模型和模板名称；
- 保留原文跳转链接。

#### 9.2.10 Markdown 导出

支持：

- 保存为 `.md` 文件；
- 用户选择保存位置；
- 文件名自动处理非法字符；
- 默认命名为 `文献标题_AI整理笔记.md`；
- 使用 UTF-8 编码。

### 9.3 MVP 不做的功能

阶段一暂不实现：

- 整篇 PDF OCR；
- 多篇文献对比；
- 思维导图；
- 自动同步 Obsidian；
- XMind 文件；
- 批量处理；
- 云端账号体系；
- 插件自建后端服务；
- 向量数据库；
- 自动识别图片和复杂公式。

### 9.4 阶段一验收标准

- 用户可以对一篇带批注的文献成功生成 Markdown；
- 所有高亮和评论均可被正确读取；
- 生成内容能区分原文和用户观点；
- 页码或 Zotero 跳转链接不丢失；
- 用户可以编辑并保存为 Zotero 笔记；
- 用户可以导出 `.md` 文件；
- API Key 不以明文显示在界面中；
- 模型调用失败时能够给出可理解的错误提示；
- 不修改原始 PDF 和原始批注。

## 阶段二：笔记模板、思维导图和 Obsidian 工作流

### 9.5 阶段目标

将产品从“批注整理工具”升级为“阅读成果加工工具”。

### 9.6 新增功能

#### 9.6.1 模板管理

支持：

- 新建模板；
- 编辑模板；
- 复制模板；
- 删除模板；
- 导入和导出模板；
- 设置默认模板；
- 编写自定义提示词；
- 配置输出章节；
- 配置不同高亮颜色的含义。

示例：

```text
黄色 = 核心观点
红色 = 风险或争议
蓝色 = 案例
绿色 = 产品启发
紫色 = 待验证问题
```

#### 9.6.2 思维导图生成

基于已生成笔记，再生成独立的树状结构。

首版推荐支持：

- Markmap Markdown；
- Mermaid mindmap；
- HTML 预览；
- SVG 导出；
- PNG 导出；
- 复制思维导图源码。

思维导图内容必须与笔记保持一致，不单独重新总结，以减少信息偏差。

#### 9.6.3 双输出模式

模型返回：

```json
{
  "markdown_note": "结构化笔记",
  "mindmap": "思维导图结构"
}
```

插件分别渲染：

- 笔记预览；
- 思维导图预览。

#### 9.6.4 Obsidian 导出

支持设置 Obsidian Vault 路径或普通 Markdown 文件夹：

- 自动导出 Markdown；
- 自定义文件名；
- 自定义目录；
- 处理同名文件；
- 可选择覆盖、创建副本或跳过；
- 支持 YAML Frontmatter。

示例：

```yaml
---
title: 文献标题
authors:
  - 作者 A
year: 2026
source: Zotero
tags:
  - AI
  - 论文精读
zotero_key: XXXXXXXX
---
```

#### 9.6.5 Prompt 预览

在发送给模型前，允许高级用户查看：

- 系统提示词；
- 文献元数据；
- 批注数据；
- 预计输入长度；
- 预计 Token 消耗。

#### 9.6.6 生成历史

记录最近生成任务：

- 文献标题；
- 生成时间；
- 使用模型；
- 使用模板；
- 是否保存；
- 是否导出；
- 失败原因。

### 9.7 阶段二验收标准

- 用户可以创建和使用自定义模板；
- 不同批注颜色可以映射为不同语义；
- 思维导图能够正确呈现多级结构；
- Markdown 与思维导图信息一致；
- 可导出到指定 Obsidian 文件夹；
- YAML Frontmatter 格式正确；
- 用户可查看模型调用所使用的数据范围；
- 生成历史可查看和再次打开。

## 阶段三：多篇文献对比、全文辅助和本地模型

### 9.8 阶段目标

支持更复杂的研究和知识整合场景。

### 9.9 新增功能

#### 9.9.1 多篇文献批注对比

用户可选择 2～10 篇文献，生成：

- 研究问题对比；
- 方法对比；
- 数据集对比；
- 实验结果对比；
- 优缺点；
- 共同结论；
- 观点冲突；
- 研究空白；
- 产品应用差异。

输出形式：

- Markdown 对比报告；
- 对比表格；
- 综合思维导图；
- Zotero 独立笔记。

#### 9.9.2 全文辅助模式

当批注不足时，用户可以主动开启：

> 使用 PDF 全文补充上下文

要求：

- 默认关闭；
- 明确提示全文会被发送给外部模型；
- 支持限制页数；
- 支持只读取摘要、目录和结论；
- 对超长文档进行分段；
- 合并结果时标记“来自全文补充”；
- 不允许把 AI 补充内容伪装成用户批注。

#### 9.9.3 本地模型支持

支持 Ollama 或其他本地 OpenAI-compatible 服务：

- 自动检测本地服务；
- 显示可用模型；
- 支持无 API Key 模式；
- 显示本地模型上下文限制；
- 提示较小模型可能导致结果质量下降。

#### 9.9.4 长文档处理

- 输入长度估算；
- 超长批注分块；
- 分块摘要；
- 二次合并；
- 保留引用映射；
- 防止不同分块内容重复。

#### 9.9.5 图片和区域批注

对于区域截图型批注：

- 提取图片；
- 可选调用视觉模型；
- 生成图表说明；
- 将图片和说明写入 Markdown；
- 保留图片来源页码。

#### 9.9.6 结果引用检查

增加简单检查机制：

- 引用内容是否存在于输入批注；
- 页码是否有效；
- 是否出现未提供的数字；
- 是否出现没有依据的作者观点；
- 标记“可能为 AI 推断”的句子。

### 9.10 阶段三验收标准

- 可选择多篇文献生成对比报告；
- 对比内容能追溯到对应文献；
- 全文模式必须经过用户主动开启；
- 本地 Ollama 模型能够完成生成；
- 超长内容不会直接导致插件崩溃；
- 区域批注可以被识别并纳入笔记；
- 系统能够标记可能缺乏依据的生成内容。

# 11. AI Prompt 设计要求

## 11.1 系统提示词原则

- 只根据提供的文献元数据、批注和评论进行整理；
- 不得伪造引用；
- 不得补充未经标记的外部知识；
- 区分原文、用户观点和 AI 归纳；
- 保留页码和链接；
- 按主题重新组织；
- 合并重复内容；
- 输出合法 Markdown；
- 信息不足时明确写“批注中未提供”。

## 11.2 推荐输出结构

为降低格式错误，模型优先返回 JSON：

```json
{
  "title": "笔记标题",
  "markdown_note": "完整 Markdown",
  "mindmap_markdown": "Markmap Markdown",
  "warnings": [
    "部分批注没有页码"
  ]
}
```

插件完成 JSON 校验后再展示内容。

## 11.3 防止幻觉

- 模型温度默认设置为较低值；
- 引用必须与输入高亮匹配；
- 生成后执行字符串相似度检查；
- 对无法确认的内容增加“AI 归纳”标记；
- 不允许自动写回，必须用户确认；
- 对数字、日期和实验结果进行重点核查。

# 12. 非功能需求

## 12.1 性能

- 打开生成窗口时间不超过 2 秒；
- 读取 500 条以内批注不应明显卡顿；
- 模型调用期间不阻塞 Zotero 主界面；
- 长任务支持取消；
- 批量任务必须异步执行。

## 12.2 稳定性

- 模型调用失败不影响 Zotero 原始数据；
- 插件崩溃后不产生损坏笔记；
- 保存前进行 Markdown 和 HTML 清理；
- 插件升级时保留用户配置。

## 12.3 隐私

- 明确说明发送给模型的数据；
- 默认只发送文献元数据和用户批注；
- 全文发送必须单独授权；
- API Key 存储在本地；
- 日志默认不记录完整文献内容；
- 支持本地模型；
- 支持清除生成历史。

## 12.4 兼容性

- 首要支持 Zotero 9；
- 支持 Windows、macOS；
- Linux 作为后续兼容目标；
- Markdown 文件采用 UTF-8；
- 中文文件名和中文路径必须正常工作。

## 12.5 易用性

- 普通用户无需理解 Prompt；
- 默认模板即可直接使用；
- 错误提示必须提供解决建议；
- 生成操作控制在 3 个步骤以内；
- 高级设置默认折叠。

# 16. 风险与应对

## 16.1 AI 幻觉

**风险：** AI 生成批注中没有的信息。

**应对：**

- 默认只基于批注；
- 低温度；
- 引用检查；
- 标记 AI 归纳；
- 用户确认后保存。

## 16.2 模型费用

**风险：** 用户批注过多导致 Token 消耗较高。

**应对：**

- 调用前估算输入长度；
- 支持选择是否发送摘要；
- 支持分块；
- 支持低价模型；
- 支持本地模型。

## 16.3 插件兼容性

**风险：** Zotero 版本升级导致 API 变化。

**应对：**

- 使用官方插件 API；
- 不直接访问数据库；
- 建立自动化测试；
- 明确支持的 Zotero 版本。

## 16.4 数据隐私

**风险：** 未公开论文或敏感资料被发送到外部模型。

**应对：**

- 首次使用时展示隐私说明；
- 默认不发送全文；
- 支持本地模型；
- 显示本次发送的数据；
- 提供“仅整理本地导出内容”模式。

## 16.5 Markdown 格式异常

**风险：** 模型输出不完整或结构错误。

**应对：**

- 优先要求 JSON；
- 结果校验；
- 自动修复常见格式；
- 支持切换为纯文本结果；
- 保留模型原始输出用于排查。

# 22. 第一阶段最小闭环

第一阶段只需确保以下闭环完整：

```text
选择一篇文献
→ 读取全部批注
→ 选择整理模板
```

只要该闭环稳定、生成质量可接受，产品就已经具备独立使用价值。思维导图、多篇对比、全文总结和知识库功能应在核心闭环经过真实用户验证后再逐步增加。
